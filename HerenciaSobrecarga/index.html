Herencia en JavaScript

Clases, extends, override y super()

Investigación Teórica

¿Qué es una clase en JavaScript?

Una clase en JavaScript es una plantilla o modelo para crear objetos. Funciona como un "molde" que define propiedades (atributos) y métodos (funciones) que tendrán los objetos creados a partir de ella. Introducidas en ES6 (ECMAScript 2015), las clases simplifican la programación orientada a objetos en JavaScript, aunque por debajo siguen usando prototipos. Por ejemplo, en esta actividad creamos la clase Producto que define propiedades como nombre, precio y stock, y métodos como mostrarDetalle().

¿Qué es la herencia (extends)?

La herencia es un principio de la programación orientada a objetos que permite crear una nueva clase basada en una clase existente. En JavaScript, usamos la palabra clave extends para establecer esta relación. La clase hija hereda todos los métodos y propiedades de la clase padre, pudiendo agregar nuevos o modificar los existentes. En nuestro ejemplo, ProductoElectronico hereda de Producto, lo que significa que automáticamente tiene acceso a nombre, precio, stock y al método mostrarDetalle() de la clase padre.

¿Qué es la sobrescritura de métodos (override)?

La sobrescritura de métodos (override) ocurre cuando una clase hija redefine un método heredado de la clase padre para darle un comportamiento específico. Esto permite mantener la misma interfaz (nombre del método) pero con una implementación diferente. En nuestra actividad, sobrescribimos el método mostrarDetalle() en ProductoElectronico para que además de mostrar los datos básicos del producto, también muestre la garantía y la marca, información específica de los productos electrónicos.

¿Para qué sirve super() en un constructor?

La función super() se utiliza en el constructor de una clase hija para llamar al constructor de la clase padre. Es obligatorio usarla antes de poder usar this en el constructor de la clase hija. Además, super también permite acceder a métodos del padre, como en super.mostrarDetalle(), que usamos para obtener los detalles base y luego agregar la información específica de garantía. Sin super(), no podríamos inicializar correctamente las propiedades heredadas.

Código Implementado

// Clase Padre
class Producto {
    constructor(nombre, precio, stock) {
        this.nombre = nombre;
        this.precio = precio;
        this.stock = stock;
    }
    
    mostrarDetalle() {
        return `Producto: ${this.nombre}, $${this.precio}`;
    }
}

// Clase Hija con herencia
class ProductoElectronico extends Producto {
    constructor(nombre, precio, stock, garantia, marca) {
        super(nombre, precio, stock);  // Llama al constructor padre
        this.garantia = garantia;
        this.marca = marca;
    }
    
    // Sobrescritura de método
    mostrarDetalle() {
        return `${super.mostrarDetalle()}, Garantía: ${this.garantia} meses, Marca: ${this.marca}`;
    }
}
